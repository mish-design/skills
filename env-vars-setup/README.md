# Environment Variables Setup

## Что делает

Настраивает **управление переменными окружения** с валидацией, TypeScript поддержкой и лучшими практиками:

- **Валидация переменных** — проверка при запуске приложения, а не в runtime
- **TypeScript типы** — автодополнение для `process.env`
- **Безопасность** — `.env.example` шаблон, реальные секреты в `.gitignore`
- **Универсальность** — поддержка Next.js, React, Vue, Node.js
- **Командная работа** — единый стандарт для всей команды

## Как работает

1. **Определяет тип проекта** (Next.js, React, Node.js, Vue)
2. **Создает `.env.example`** — шаблон с комментариями для всех переменных
3. **Настраивает валидацию** через `zod` схему
4. **Добавляет TypeScript типы** для автодополнения
5. **Обновляет `.gitignore`** — защищает секреты
6. **Добавляет скрипты** в `package.json` для удобства

## Особенности

### Для разных типов проектов:
- **Next.js**: Автоматическая загрузка `.env.local`, префикс `NEXT_PUBLIC_*`
- **React/Vite**: Префикс `VITE_*`, `import.meta.env`
- **Node.js**: Установка `dotenv`, загрузка в entry point
- **TypeScript**: Полная типизация `process.env`

### Валидация:
```typescript
const envSchema = z.object({
  DATABASE_URL: z.string().url(),
  NEXTAUTH_SECRET: z.string().min(32),
  NODE_ENV: z.enum(['development', 'production']),
});
```

### Безопасность:
- `.env.example` — безопасно коммитить (шаблон)
- `.env.local` — в `.gitignore` (реальные значения)
- Проверка формата ключей (Stripe, OpenAI и др.)

## Созданные файлы

### `.env.example` (безопасно коммитить)
```bash
# Шаблон с комментариями
DATABASE_URL="postgresql://user:password@localhost:5432/dbname"
NEXTAUTH_SECRET="your-secret-key-here"
NEXT_PUBLIC_API_URL="http://localhost:3000/api"
```

### `src/env.ts` (валидация)
```typescript
import { z } from 'zod';
const envSchema = z.object({ ... });
export const env = envSchema.parse(process.env);
```

### `env.d.ts` (TypeScript типы)
```typescript
declare global {
  namespace NodeJS {
    interface ProcessEnv extends z.infer<typeof envSchema> {}
  }
}
```

### `package.json` (скрипты)
```json
{
  "scripts": {
    "env:validate": "node -r ./src/env.ts",
    "env:check": "node scripts/setup-env.js",
    "env:example": "cp .env.example .env.local"
  }
}
```

## Безопасность

### ✅ Правильно:
- `.env.example` в репозитории (шаблон)
- `.env.local` в `.gitignore`
- Разные ключи для development/production
- Использование secrets managers в production

### ❌ Неправильно:
- Реальные API ключи в репозитории
- Одинаковые ключи для всех окружений
- Жестко закодированные секреты в коде
- `.env.production` в репозитории

## Поддерживаемые форматы переменных

### API ключи:
- Stripe: `sk_test_...`, `pk_test_...`
- OpenAI: `sk-...`
- SendGrid: `SG.`

### URL:
- Базы данных: `postgresql://...`, `mongodb://...`
- API endpoints: `https://api.example.com`
- Frontend URLs: `http://localhost:3000`

### Секреты:
- JWT secrets: минимум 32 символа
- Encryption keys: в правильном формате
- Пароли: не в plain text

## Проблемы и решения

### Проблема: "Переменные не загружаются"
**Решение**: Проверить:
1. Существует ли `.env.local`
2. Правильное имя переменной
3. Перезапустить dev server (env vars кэшируются)

### Проблема: "TypeScript ошибки"
**Решение**: Убедиться, что:
1. `env.d.ts` включен в `tsconfig.json`
2. Типы правильно экспортируются
3. Нет конфликтов с другими типами

### Проблема: "Валидация падает в production"
**Решение**: 
1. Проверить переменные на хостинге
2. Использовать `.env.production` для production значений
3. Добавить значения по умолчанию для необязательных переменных
