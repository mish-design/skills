# Environment Variables Setup

## Что делает

Базовый скилл для настройки переменных окружения с валидацией и TypeScript поддержкой:

- **Создает `.env.example`** - шаблон с комментариями
- **Настраивает валидацию** через `zod`
- **Добавляет TypeScript типы** для `process.env`
- **Создает скрипты** в `package.json`
- **Гарантирует безопасность** - `.env.local` в `.gitignore`

## Что именно настраивается

- **`.env.example`**: шаблон переменных окружения (его можно коммитить)
- **`.gitignore`**: правило `.env*` + исключение `!.env.example`, чтобы секреты не попадали в git
- **`src/env.js`**: рантайм-валидация `process.env` через `zod` (fail-fast)
- **`env.d.ts`** (если TypeScript): базовые типы для `process.env`
- **`package.json`** (опционально): удобные скрипты `env:init` и `env:check`

## Установка

```bash

```

## Что создается

1. **`.env.example`** - шаблон переменных окружения
2. **`src/env.js`** - рантайм-валидация через `zod` (fail-fast)
3. **`env.d.ts`** - TypeScript типы (для TS проектов)
4. **Скрипты** в `package.json` (опционально) - `env:init`, `env:check`

## Использование

### 1. После установки:
```bash
cp .env.example .env.local
```

### 2. Отредактируйте `.env.local`:
```bash
# Откройте файл и замените значения
DATABASE_URL="postgresql://realuser:realpass@localhost:5432/realdb"
```

### 3. Проверьте валидацию:
```bash
npm run env:validate
```

## Безопасность

### ✅ **Правильно:**
- `.env.example` в репозитории (шаблон)
- `.env.local` в `.gitignore`
- Разные ключи для development/production

### ❌ **Неправильно:**
- Реальные ключи в репозитории
- `.env.local` в коммитах
- Одинаковые ключи для всех окружений

## Проблемы и решения

### Ошибки валидации:
1. Перезапустите dev server после изменения `.env`
2. Проверьте имена переменных в схеме
3. Убедитесь что `.env.local` существует

## Для разных фреймворков

### Next.js
- Автоматически загружает `.env.local`
- Используйте `NEXT_PUBLIC_` префикс для переменных браузера

### Vite/React/Vue
- Используйте `VITE_` префикс для переменных браузера
- Доступ через `import.meta.env.VITE_*`

### Node.js
- Установите `dotenv` пакет
- Загрузите: `require('dotenv').config()`
