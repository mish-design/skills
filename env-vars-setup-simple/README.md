# Environment Variables Setup (Simple Version)

## Что делает

Упрощенная версия скилла для настройки переменных окружения с валидацией и TypeScript поддержкой:

- **Создает `.env.example`** - шаблон с комментариями
- **Настраивает валидацию** через `zod`
- **Добавляет TypeScript типы** для `process.env`
- **Создает скрипты** в `package.json`
- **Гарантирует безопасность** - `.env.local` в `.gitignore`

## Особенности простой версии

### ✅ **Преимущества:**
- **Один файл** - только `SKILL.md`, нет зависимостей
- **Быстрая установка** - меньше проблем с правами доступа
- **Проще для систем навыков** - избегает ошибок EPERM
- **Базовые функции** - все необходимое для старта

### ⚠️ **По сравнению с полной версией:**
- Нет шаблонов для разных фреймворков
- Более простая детекция проекта
- Меньше обработки ошибок
- Нет advanced функций

## Когда использовать

### ✅ **Используйте простую версию если:**
- Возникают ошибки `EPERM` при установке полной версии
- Нужен быстрый старт без сложностей
- Проект небольшой или стандартный
- Не нужны продвинутые функции

### 🔧 **Используйте полную версию если:**
- Нужны шаблоны для Next.js, Vite, Node.js
- Требуется продвинутая обработка ошибок
- Нужна лучшая кроссплатформенность
- Требуются дополнительные функции

## Установка

```bash
npx skills add https://github.com/mish-design/skills --skill env-vars-setup-simple
```

## Что создается

1. **`.env.example`** - шаблон переменных окружения
2. **`src/env.ts`** или **`src/env.js`** - валидация
3. **`env.d.ts`** - TypeScript типы (для TS проектов)
4. **Скрипты** в `package.json` - `env:validate`, `env:init`

## Использование

### 1. После установки:
```bash
cp .env.example .env.local
```

### 2. Отредактируйте `.env.local`:
```bash
# Откройте файл и замените значения
DATABASE_URL="postgresql://realuser:realpass@localhost:5432/realdb"
```

### 3. Проверьте валидацию:
```bash
npm run env:validate
```

## Безопасность

### ✅ **Правильно:**
- `.env.example` в репозитории (шаблон)
- `.env.local` в `.gitignore`
- Разные ключи для development/production

### ❌ **Неправильно:**
- Реальные ключи в репозитории
- `.env.local` в коммитах
- Одинаковые ключи для всех окружений

## Проблемы и решения

### Ошибка `EPERM`:
Если полная версия вызывает ошибки прав доступа, используйте эту упрощенную версию.

### Ошибки валидации:
1. Перезапустите dev server после изменения `.env`
2. Проверьте имена переменных в схеме
3. Убедитесь что `.env.local` существует
